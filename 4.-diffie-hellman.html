01. El Problema de la Clave Compartida


  • Cifrado César: Un método clásico donde cada letra se desplaza 'n' posiciones.
  • Para que Alice y Bob se comuniquen, ambos deben conocer el valor de 'n' (la clave).

⚠️ Problema fundamental: ¿Cómo comparten esa clave 'n' de forma segura sin que un tercero la intercepte?

Si Alice envía la clave por email, un atacante (Eve) que intercepte el correo, leerá absolutamente todo.

02. El Desafío: El atacante escucha


Imagina que estamos en una red pública (internet). Todo lo que envías puede ser "visto" por otros.

> Traceroute: intercepting packets...
> Target A <---> Target B fully visible

¿Podemos acordar un secreto compartido si el canal de comunicación es totalmente público y transparente?

Cualquier algoritmo que requiera que ambos "sepan" la clave de antemano es vulnerable si alguien escucha el proceso de intercambio.

03. La Idea Central

Propuesto por Whitfield Diffie y Martin Hellman en 1976.

No necesitamos enviarnos la clave secreta.

Ambas partes pueden construir una clave compartida a partir de información pública.

🎨 Analogía: Es como mezclar pinturas: es fácil mezclar colores, pero casi imposible separar la mezcla original para volver a un solo color inicial.

04. Magia de la Aritmética Modular

El algoritmo se basa en la dificultad de resolver el Problema del Logaritmo Discreto.

1. Acuerdan datos públicos: primo grande p y base g.

2. Alice elige a oculto. Calcula: A = ga mod p

3. Bob elige b oculto. Calcula: B = gb mod p

4. Intercambian A y B en público.

5. Alice calcula (B)a mod p = K

6. Bob calcula (A)b mod p = K

¡Ambos terminan con el mismo número K, el cual nunca fue enviado por la red!

05. Ejemplo Numérico Práctico

Parámetros públicos (vistos por todos): p = 23, g = 5

# Lado de Alice (Secreto a = 6)
56 mod 23 = 15625 mod 23 = 8 // Alice envía 8

# Lado de Bob (Secreto b = 15)
515 mod 23 = 19 // Bob envía 19

# Generación de la Clave Secreta Compartida
Alice recibe 19 -> 196 mod 23 = 2
Bob recibe 8 -> 815 mod 23 = 2

06. ¿Por qué no lo descifra el atacante?

El atacante (Eve) en el medio de la red solo ve:
p=23, g=5, A=8, B=19.

// Para romperlo, Eve tendría que resolver:
5^a = 8 mod 23
  • En nuestro ejemplo pequeño, adivinar "a=6" es fácil.
  • En la vida real, 'p' tiene cientos (o miles) de dígitos.
  • Logaritmo discreto: No existe un algoritmo eficiente para revertir esta operación en un tiempo razonable, ni con súpercomputadoras.
